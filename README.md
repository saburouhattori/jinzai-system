# Smile Dream 人材事業 総合管理システム

## 概要
本システムは、Smile Dream Co.,Ltd. が自社運営する、ミャンマーを中心とした海外人材の登録、案件（求人）へのマッチング、採用後の進捗管理を一元的に行う Google Apps Script (GAS) アプリケーションです。
スプレッドシートをデータベースとして活用し、カスタムサイドバー（HTML/JS）を通じて直感的なデータ操作を可能にしています。
また、主要な取引先である株式会社Funtocoが管理する外部スプレッドシート（海外ミャンマー管理リスト等）へのデータ自動連携機能を備えています。

## ファイル構成と主要機能
- **`01_共通・メニュー.gs`**
  - システムの基盤。各シート名やスプレッドシートIDの定義、カスタムメニューおよびサイドバーの表示制御を担当。
- **`02_候補者管理.gs`**
  - `登録者マスタ` シートへのCRUD操作。
- **`03_案件・採用管理.gs`**
  - `案件管理` シートおよび `採用者一覧` へのデータ反映ロジック。
- **`04_事業者管理.gs`**
  - `事業者マスタ` シートの管理。
- **`05_マスタ連携・その他.gs`**
  - 送り出し機関や日本語学校といった外部マスタとの紐付け処理。
- **`06_外部連携.gs`**
  - 取引先（Funtoco等）の外部システムとのデータ同期・転記処理。
- **`07_履歴書作成.gs`**
  - スプレッドシートをテンプレートとした履歴書PDFおよびシートの自動生成。
- **UI系 (`*.html`)**
  - サイドバー用のHTML、CSS、クライアントサイドJavaScript。

## スプレッドシート構成（主要シート）
- **登録者マスタ**: 人材の基本情報、学歴、職歴、日本語能力などを管理。
- **案件管理**: 各求人のステータス、事業者名、面接日、採用者リストを管理。
- **事業者マスタ**: 受け入れ先企業の情報。

## 外部連携機能について（取引先：Funtoco「支払い管理」シートへの同期）
本システム（Smile Dream側）で確定した採用情報を、取引先であるFuntocoの「海外ミャンマー管理リスト」内にある「支払い管理」シートへ自動で連携・同期します。（処理ファイル: `06_外部連携.gs`）

* **連携元 (Smile Dream側)**: `案件管理` シート
* **連携先 (Funtoco側)**: Funtoco管理 `海外ミャンマー管理リスト` の `支払い管理` シート
* **連携の仕組みと仕様**:
  1. **データの取得と展開**:
     `案件管理` シートにて採用者が確定し、「採用者名」列に `SD-0064-HNIN EI HLAING` のような形式（改行区切りで複数人可）で入力されると、スクリプトがこれを個別の登録者IDと名前にパースして展開します。
  2. **データの同期**:
     システムメニューから同期を実行すると、該当の採用者情報（案件ID、登録者ID、事業者名、技能分野、名前）がFuntoco側の「支払い管理」シートへ自動的に追記・更新されます。
  3. **データ保護（安全設計）**:
     Funtoco側で既に手入力されている「ステータス」「入国日」「入職日」「金額」「備考」などの業務データは上書きされず、強固に保護される設計になっています（Smile Dream側の内定日等による上書きは行いません）。
  4. **重複チェック・警告機能**:
     同期時、同一の登録者IDが「Funtoco側のシート内で既に別の案件IDで登録されている」ことを検知した場合、同期完了のポップアップにて警告メッセージと該当者リストが表示されます。警告が出た場合は、運用担当者がFuntoco側の「支払い管理」シートで古い不要なデータの確認・整理を行う運用としています。

### 外部連携のセットアップ
外部連携先のシートIDは `06_外部連携.gs` 内の定数 `EXTERNAL_SS_ID_FUNTOCO` に定義されています。連携先シートが変更になった場合は、このIDを修正してください。

## 開発・運用ガイドライン
- **破壊的変更の禁止**: 既存のシート構造やIDマッピングに依存している処理が多いため、列の挿入・削除を行う際は `getMasterColumnMap` 等のマッピングロジックが正常に追従できるか確認してください。
- **GASの制限**: 6分の実行時間制限を考慮し、大量のデータをループ処理する場合は `getValues()` や `setValues()` を用いた一括処理（バッチ処理）を徹底してください。
