# 人材管理システム (Jinzai System)

## 概要 (Overview)

Google Apps Script (GAS) と Google スプレッドシートを活用した人材紹介事業（Smile Dream Co.,Ltd.）向けの管理システムです。
候補者（登録者）の管理、求人案件の管理、面接から採用（内定）までのプロセスを一元管理し、業務の効率化を図ります。日本およびミャンマー拠点のスタッフが共同で運用することを想定して設計されています。

## 主な機能 (Features)

* **候補者管理**:
  「登録者マスタ」への新規登録、更新、削除。顔写真データを含んでいても高速に一括更新可能なロジックを実装。

* **年齢自動計算**:
  スプレッドシートの DATEDIF 関数と ARRAYFORMULA を活用し、生年月日から満年齢をリアルタイムに自動計算。メンテナンスフリーな運用を実現。

* **案件・採用管理**:
  求人案件の登録、修正、削除機能。および案件に対する候補者の「採用（内定）」登録機能。

* **面接履歴の自動追記**:
  「採用者登録」実行時に、該当案件で面接を受けた全候補者（採用・不採用問わず）の「面接履歴」と結果を、「登録者マスタ」へ自動で改行追記します。

* **自動リスト同期 (動的マッピング)**:
  登録者マスタの更新内容を「採用者一覧」「未採用者一覧」「簡易リスト」へ自動同期。ヘッダー名（1行目の文字列）をキーにしてデータを自動振り分けするため、列の追加や移動にも柔軟に追従します。

* **直感的なUI / 待機表示**:
  データの送受信や同期など、時間のかかる非同期処理中には画面全体を覆うローディングスピナーを表示し、システムの状態を視覚的にフィードバックします。

* **ファイルリンク連携**:
  関連ファイルURLをGoogleドライブのファイル名を取得して「スマートチップ風」に自動変換。

* **履歴書・簡易リスト生成**:
  候補者IDを指定して、提出用の履歴書や簡易リストを自動生成する機能。

* **支払い管理連携（外部連携）**:
  取引先である株式会社Funtoco様の外部シート「海外ミャンマー管理リスト【Funtoco】」へ、案件管理で確定した採用者情報をボタン一つで同期。
  同一登録者が別の案件で既に登録されている場合の重複警告機能を搭載し、Funtoco側で入力された入国日・金額等の業務データは保護しつつ、基本情報のみを安全に更新可能。転記後は自動で「案件ID順」にソートされます。

## システム構成 (System Structure)

### スプレッドシート（データベース）

* **登録者マスタ**: 全候補者のベースデータ（ID: SD-XXXX）
* **案件管理**: 求人案件データ（ID: JOB-XXXX）
* **リスト・出力用**: 未採用者一覧, 採用者一覧, 簡易リスト, 履歴書, 面接管理
* **各種マスタ**: 事業者マスタ, 送り出し機関マスタ, 日本語学校マスタ, 企業情報, 面談用質問

### Google Apps Script (バックエンド・フロントエンド)

* 01_共通・メニュー.gs / JS_Common.html : 共通処理・UIの制御・メニュー構築
* 02_候補者管理.gs / HTML_Candidate.html / JS_Candidate.html : 候補者データ操作
* 03_案件・採用管理.gs / HTML_Job.html / JS_Job.html : 案件データ操作・採用登録
* 04_事業者管理.gs / HTML_Company.html : 事業者データ操作
* 05_マスタ連携・その他.gs : リストの自動同期（syncListSheets）、外部シート連携、辞書生成などの補助機能
* 06_外部連携.gs : 取引先（Funtoco等）の外部システムとのデータ同期・転記処理
* 07_履歴書作成.gs : 履歴書の自動出力処理
* MainSidebar.html / Sidebar_CSS.html : サイドバーのUIとスタイリング

### 【今後の改修・アップデート用メモまとめ】

**1. 入力フォームのUI改善（プレースホルダーと文字数制限）**
* **目的:** ユーザーの入力ミスを減らし、想定外の長文が登録されるのを防ぐ。
* **実装方法（HTML側の修正）:**
    * 文面の変更: `<input placeholder="新しい入力例">` に書き換える。
    * 文字数制限: `<input maxlength="20">`（20文字制限の場合）をタグ内に追加する。

**2. 「年月のみ」の日付データの扱い（日にち不要の項目）**
* **目的:** 職歴や学歴など、日にちの入力が不要な項目の入力しやすさと、スプレッドシート上でのデータ形式を最適化する。
* **実装方法:**
    * 入力側: `<input type="text">` を `<input type="month">` に変更し、カレンダー（年月ピッカー）から選ばせる。
    * 保存・出力側: スプレッドシートが自動で「◯月1日」と補完してしまうのを防ぐため、GAS側で「2026年3月」という単なる**文字列（テキスト）**として保存させる。または、履歴書出力時にGASの `Utilities.formatDate()` で年月のみに変換して出力する。

**3. 数字の表記ゆれ防止（「03月」「3月」「３月」など）**
* **目的:** データ検索や履歴書の出力品質を保つため、人による入力表記のバラつきをなくす。
* **実装方法:**
    * UIでの防止: 上記の `<input type="month">` や、年・月のプルダウン（`<select>`）を採用し、そもそも自由に入力させない形にするのが最も確実。
    * プログラムでの補正: 自由入力を残す場合は、GASの保存処理の直前に「全角数字を半角数字に変換する」「03を3に（または3を03に）統一する」という自動変換プログラムを組み込む。

**4. 辞退者のアーカイブ化**
* ステータスが「辞退」の候補者のアーカイブ化処理を追加する。

**5. データクリーニング（空白文字の補正）**
* 登録者の名前やフリガナに意図しない空白が混入しているため、登録・更新時に自動でトリム（空白除去）するロジックを追加する。
