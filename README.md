# Funtoco 人材事業 総合管理システム

## 概要
本システムは、ミャンマーを中心とした海外人材の登録から、案件（求人）へのマッチング、採用後の進捗、および外部スプレッドシート（支払い管理など）への連携を一元管理する Google Apps Script (GAS) アプリケーションです。
スプレッドシートをデータベースとして活用し、カスタムサイドバー（HTML/JS）を通じて直感的なデータ操作を可能にしています。

## ファイル構成と主要機能
- **`01_共通・メニュー.gs`**
  - システムの基盤。各シート名やスプレッドシートIDの定義、カスタムメニューおよびサイドバーの表示制御を担当。
- **`02_候補者管理.gs`**
  - `登録者マスタ` シートへのCRUD操作。
- **`03_案件・採用管理.gs`**
  - `案件管理` シートおよび `採用者一覧` へのデータ反映ロジック。
- **`04_事業者管理.gs`**
  - `事業者マスタ` シートの管理。
- **`05_マスタ連携・その他.gs`**
  - 送り出し機関や日本語学校といった外部マスタとの紐付け処理。
- **`06_外部連携.gs`**
  - 外部システム（海外ミャンマー管理リスト等）とのデータ同期。
- **`07_履歴書作成.gs`**
  - スプレッドシートをテンプレートとした履歴書PDFおよびシートの自動生成。
- **UI系 (`*.html`)**
  - サイドバー用のHTML、CSS、クライアントサイドJavaScript。

## スプレッドシート構成（主要シート）
- **登録者マスタ**: 人材の基本情報、学歴、職歴、日本語能力などを管理。
- **案件管理**: 各求人のステータス、事業者名、面接日、採用者リストを管理。
- **事業者マスタ**: 受け入れ先企業の情報。

## 外部連携機能について（支払い管理への同期）
本システムは、外部の「海外ミャンマー管理リスト」とデータを連携しています。現在は `06_外部連携.gs` の `syncToPaymentManagement()` によって処理されます。

* **連携元**: `案件管理` シート
* **連携先**: `海外ミャンマー管理リスト` の `支払い管理` シート
* **連携の仕組みと仕様**:
  1. **データの取得と展開**:
     `案件管理` シートにて採用者が確定し、「採用者名」列に `SD-0064-HNIN EI HLAING` のような形式（改行区切りで複数人可）で入力されると、スクリプトがこれを個別の登録者IDと名前にパースして展開します。
  2. **データの同期**:
     システムメニューから同期を実行すると、該当の採用者情報（案件ID、登録者ID、事業者名、技能分野、名前）が「支払い管理」シートへ自動的に追記・更新されます。
  3. **データ保護（安全設計）**:
     「支払い管理」シート側に既に手入力されている「ステータス」「入国日」「入職日」「金額」「備考」などのデータは上書きされず、保護される設計になっています（内定日の自動転記は行いません）。
  4. **重複チェック・警告機能**:
     同期時、同一の登録者IDが「すでに別の案件IDで登録されている」ことを検知した場合、同期完了のポップアップにて警告メッセージと該当者リストが表示されます。警告が出た場合は、運用担当者が「支払い管理」シートにて古い不要なデータの確認・整理を行う運用としています。

### 外部連携のセットアップ
外部連携先のシートIDは `06_外部連携.gs` 内の定数 `EXTERNAL_SS_ID_FUNTOCO` に定義されています。連携先が変更になった場合は、このIDを修正してください。

## 開発・運用ガイドライン
- **破壊的変更の禁止**: 既存のシート構造やIDマッピングに依存している処理が多いため、列の挿入・削除を行う際は `getMasterColumnMap` 等のマッピングロジックが正常に追従できるか確認してください。
- **GASの制限**: 6分の実行時間制限を考慮し、大量のデータをループ処理する場合は `getValues()` や `setValues()` を用いた一括処理（バッチ処理）を徹底してください。
